## 五、上线阻断与风险

### 高风险：机器人回复当前关闭

当前配置为 `FEISHU_REPLY_ENABLED=false`。长连接收到口令后，`sendReply` 会记录“跳过回复”，用户无法收到帮助、领取结果、完成结果或错误信息。上线前应确认业务是否允许静默执行；若不允许，必须开启回复并在测试群复测全部口令。

### 高风险：长连接必须独立常驻

仅启动 Web 服务时，健康检查返回“飞书长连接心跳超时”。执行 `npm run feishu:ws` 后恢复正常。上线部署必须同时守护 Web 服务与长连接进程，并配置自动重启、启动顺序及健康告警。

### 中风险：管理操作依赖原生 `prompt()`

“测试连接”“读取字段”会再次调用 `window.prompt()` 获取配置写入密码。当前内置浏览器不支持该 API，控制台出现 `prompt() is not supported`，导致按钮操作无法完成。建议改为站内密码弹窗或复用已登录的管理员会话，再做 Chrome/Edge 复测。

### 覆盖缺口：文件选择器与真实飞书群消息

自动化环境未完成文件选择器上传；写入解析和真实双表写入已通过接口验证。真实飞书群消息还需要指定隔离测试群后，逐条验证 8 个口令及回复内容。

## 六、上线前建议清单

1. 开启并确认 `FEISHU_REPLY_ENABLED` 的预期值；在隔离测试群验证 8 条机器人口令。
2. 使用进程守护同时运行 `npm run server` 与 `npm run feishu:ws`，验证机器重启后自动恢复。
3. 将 `window.prompt()` 改为站内弹窗或管理员会话鉴权，复测“测试连接、读取字段、保存配置/映射”。
4. 用一份最小 CSV 和一份最小 XLSX 从页面文件选择器分别导入胶板、油漆，再按返回记录 ID 清理。
5. 完成以上项目后重新执行健康检查、页面冒烟和数据闭环，再给出正式上线放行结论。
